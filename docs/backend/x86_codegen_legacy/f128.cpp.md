# `f128.cpp`

Primary role: long-double (`f128`) lowering and x87-backed storage/load support.

Key surfaces:

```cpp
std::optional<std::int64_t> X86Codegen::emit_f128_resolve_addr(const SlotAddr&, std::uint32_t, std::int64_t) const;
void X86Codegen::emit_f128_fldt(const SlotAddr&, std::uint32_t ptr_id, std::int64_t offset);
void X86Codegen::emit_f128_store_f64_via_x87(const SlotAddr&, std::uint32_t ptr_id, std::int64_t offset);
```

- Concentrates the x87-specific addressing and store/load sequences that do not fit the normal SSE-based float path.
- Reuses `SlotAddr` and frame-address helpers from the canonical memory stack.
- Represents a durable backend capability, but one with strong legacy-architecture pressure.
- Special-case classification: `core lowering` with `legacy compatibility` pressure from x87-only behavior.
