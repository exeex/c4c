# Prepared Target Consumer Boundary Audit

Status: Drafted for idea 418 Step 2
Source Idea: `ideas/open/418_prepared_target_consumer_boundary_audit.md`

This audit classifies RV64/RISC-V and AArch64 target prepared consumers after
the idea 413 prepared contract taxonomy. The rows below distinguish coherent
lowering over complete prepared facts from target-side recovery of facts that
should be producer-owned. Ideas 414 through 417 should cite these audit rows
instead of re-running the same target-consumer search.

## Classification Rules

- Acceptable coherent-fact lowering means the target consumes complete prepared
  facts and only chooses a target instruction form, copy order, or supported
  emission shape.
- Suspected target recovery means the target reconstructs ABI routes, stack
  offsets, symbol names, initializer bytes, publication state, or source
  registers from adjacent BIR/source/prepared fragments instead of consuming a
  single complete prepared contract fact.
- Missing or contradictory producer facts remain `producer_missing` or
  `producer_incoherent`; the target should fail closed instead of guessing.
- Coherent but currently unsupported target routes remain
  `target_unsupported_but_coherent`.

## Step 3 Candidate Decision

The clearest low-risk Step 3 owner decision is
`418-AUD-RV64-OBJECT-GLOBAL-RECOVERY-001` plus
`418-AUD-RV64-GLOBAL-MEMORY-RECOVERY-001`: RV64 currently derives labels and
initializer bytes from raw BIR globals/string constants while the taxonomy
reserves prepared global initializer, object-data publication, and memory
access facts. If Step 3 chooses a documentation-only owner decision, route this
work to Idea 417 with explicit reject signals for raw initializer-byte
reconstruction and fallback symbol spelling in target code.
