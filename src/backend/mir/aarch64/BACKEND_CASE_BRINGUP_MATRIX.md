# AArch64 Backend Case Bring-Up Matrix

This matrix is the durable routing artifact for AArch64 public backend case
bring-up. A case may move from deferred or blocked to enabled only after its
markdown owner has a structured owner in `.cpp/.hpp`, dependent machine-node
or natural operator coverage, and public smoke expectation.

## Reject Rules

- Reject testcase-overfit: do not add fixture-shaped MIR, codegen, printer, or
  selection shortcuts whose main effect is making one named case pass.
- Reject expectation downgrade: do not weaken a supported public case, mark it
  unsupported, or replace external smoke expectation with a looser assertion to
  claim progress.
- Keep enabled public cases on the `c4cll --codegen asm --target
  aarch64-linux-gnu` route and use external assembler/compiler proof where the
  case is intended to assemble and run.
- Keep internal assembler, encoder, object writer, linker, and binary output
  out of this matrix. They are not backend case input routes.

## Status Vocabulary

- `enabled`: wired public AArch64 case coverage exists today.
- `deferred`: the case is a valid future AArch64 target, but its markdown owner
  still needs structured implementation coverage before public enablement.
- `blocked`: the case family needs a missing ABI, carrier, relocation, or other
  architectural contract before a narrow feature packet can honestly enable it.

## First Next Feature Group

The first next feature group should be scalar return/ALU:

```text
src/backend/mir/aarch64/codegen/alu.md
  + src/backend/mir/aarch64/codegen/returns.md
  -> src/backend/mir/aarch64/codegen/{alu,returns,instruction,operands,dispatch}.*
  -> scalar selected machine-node/operator coverage and temporary terminal printer
  -> tests/backend/case/return_add.c
```

That route is small enough to prove semantically without testcase-overfit. It
should add or strengthen focused C++ record/printer coverage first, then enable
public `return_add` smoke only when selected `add` and `ret` output is produced
through the normal backend path. `return_add_sub_chain` and two-argument ALU
fixtures should remain deferred until the same route handles them naturally.
