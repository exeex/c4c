# `x86.hpp` contract

## Legacy Evidence

The old root x86 surface was split across `x86_codegen.hpp` and
`route_debug.hpp`, and both acted as compatibility umbrellas for a much larger
legacy implementation.

## Design Contract

`x86.hpp` is now the only root x86 header.

Owned outputs:

- thin compatibility wrappers for route summary and trace
- thin compatibility wrappers for symbol and private-label spelling

Must not own:

- module emission
- debug implementation
- ABI policy beyond forwarding to `abi/`
- any stateful codegen surface
